That error means something inside the chart is still trying to use the **default Y-axis id `0`**. Our chart only defined custom ids (`rate`, `bp`, `spo2`, `temp`), so Recharts can’t find `0` and throws.

The simplest, safe fix is to **add a hidden Y-axis with id `0`** so anything that defaults to `0` (e.g., an internal component or a future series you add without a yAxisId) has a home.
