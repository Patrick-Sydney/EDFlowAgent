Awesome—let’s make the **desktop Patient Card** cleaner by **removing collapsed CTAs** and giving that space to **status/alert symbols**.

Below is a single patch that:

* Hides the collapsed **CTA button on desktop** (keeps it on mobile).
* Adds a tiny **StatusStrip** (isolation, allergy, oxygen, sepsis alert, tasks due, results pending).
* Leaves your existing expanded actions intact.

Save this as `desktop-card-no-cta.patch` and apply it.

## How to use this in Replit

1. **Open Shell**, make a branch, apply:

   ```bash
   git checkout -b ui/desktop-card-no-cta
   printf "%s\n" 'PASTE THE PATCH ABOVE HERE' > desktop-card-no-cta.patch
   git apply --reject --whitespace=fix desktop-card-no-cta.patch
   git add -A
   git commit -m "Desktop: hide collapsed CTAs; add StatusStrip area for insights"
   ```

2. **Pass status flags** when rendering desktop cards (one place, all lanes).
   In your **desktop board** where you map patients to `PatientCardExpandable`, add `statusFlags`:

   ```tsx
   <PatientCardExpandable
     // ...existing props
     statusFlags={{
       isolation: p.isolation,           // from Flag/precautions
       allergy: !!(p.allergies?.length),
       oxygen: p.onOxygen,               // from latest obs/orders
       sepsis: p.sepsisPathway === true, // or from alerts
       tasksDue: p.tasksDueCount,        // number
       resultsPending: p.resultsPending, // number
     }}
   />
   ```

   That’s it—**no CTA on desktop** (it’s hidden with `md:hidden`), and the **status strip** appears where the button used to live.

3. **Mobile stays unchanged.** The CTA still shows on phones (RN “Start Triage”, etc.), because we only hid it on `md+`.
